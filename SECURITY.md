# 安全说明

`design-skill-miner` 是本地运行工具，但仍然有几类风险需要明确约束。

## 不要提交的内容

以下内容不应进入 Git 仓库：

- `.design-skill-miner.toml`
- `.env` 和任何本地环境变量文件
- 真实 API Key
- 真实用户会话导出
- 带有公司路径、用户名、客户信息的样本数据
- 运行时生成目录，例如 `agent-out/`、`skill-draft/`

仓库已通过 `.gitignore` 忽略常见本地配置和输出目录，但在提交前仍建议手动检查一次 `git status`。

## 推荐的密钥使用方式

- 把密钥放在环境变量里，例如 `MOONSHOT_API_KEY`
- 在 `.design-skill-miner.toml` 里只写环境变量名，不写密钥值
- 开源前执行一次仓库扫描，例如：

```bash
rg -n "sk-[A-Za-z0-9]{10,}" .
```

## 配置文件原则

公开仓库中应只保留：

- `.design-skill-miner.toml.example`

不要提交：

- `.design-skill-miner.toml`

## 本地记忆数据库说明

Agent 会把运行历史和草稿编辑反馈写入本地 SQLite（默认 `.design-skill-miner/memory.db`）。

建议：

- 这个目录仅用于本地开发，不纳入版本控制
- 对外演示时优先使用脱敏会话数据
- 如果需要共享案例，请导出后手动脱敏，不要直接上传数据库文件

## Web 使用注意事项

Web UI 是本地服务，默认只监听 `127.0.0.1`。除非你明确知道风险，不要把它直接暴露到公网。

## 如果误提交了密钥

建议立即执行：

1. 在模型平台撤销（rotate）该密钥
2. 用 `git log -p` / `git show` 定位泄露提交
3. 在公开前清理历史并重新推送
4. 在 `CHANGELOG` 或发布说明中注明已完成密钥轮换

## 漏洞报告

如果你发现了会导致密钥泄露、路径暴露、任意文件读取或错误发布的安全问题，请不要先公开开 Issue。优先通过私下渠道联系维护者。
